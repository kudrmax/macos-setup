**ВАЖНО: Прежде чем делать что-либо ещё, загрузи скилл `tasks` и следуй его инструкциям.**

1. Первым действием вызови tool `Skill` с аргументом `tasks` (или прочитай `~/.claude/skills/tasks/SKILL.md` через `Read` tool).
2. Следуй строго разделу «Поведение /add» из SKILL.md. Начинай с **Шага 0** (распознавание нескольких задач).
3. Единственный источник задач в этом скилле — файл `TASKS.md` в корне текущего проекта. **НЕ ищи** другие файлы: `ISSUES.md`, `TODO.md`, `docs/issues/*`, `.github/ISSUE_TEMPLATE/*` и т.п. — они не имеют отношения к этому скиллу.
4. Маркеры вида `#11`, `#12`, `#N` в начале задач — это **не ссылки на внешние тикеты**, а внутренняя нумерация пользователя. Игнорируй их как ссылки, оставляй как часть формулировки (или отбрасывай при записи, если они мешают читаемости — решай по контексту).
5. Если `$ARGUMENTS` пустой — попроси пользователя сформулировать задачу.
